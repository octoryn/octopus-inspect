# 安全策略

## 报告漏洞

安全漏洞请**不要提交公开 issue**。

请通过 GitHub Security Advisories(仓库 Security 页的 "Report a vulnerability")
或邮件 **security@octopusos.ai** 私下报告。请包含描述、复现步骤与影响。我们力争在
数个工作日内确认。

## 范围说明

Inspect 是一个**静态治理 linter**:它读取磁盘上的工作区并报告发现项。按设计,以下
几处与安全相关:

- **不可信输入。** Inspect 会在任意、可能带有敌意的文件上运行。面对畸形内容它**绝不
  崩溃** —— 一个无法解析为 JSON 的文件会被跳过,而非致命错误;在任意目录树上运行都
  返回一份报告,而非抛异常。若发现任何导致抛异常、卡死或原型污染的输入,请报告。
- **不执行、不出站。** 扫描过程中,Inspect **不会**执行工作区、导入运行时、跟随
  符号链接,也不做任何网络 I/O。唯一的例外是**插件 (plugin)**:插件是 Inspect
  `import` 并运行其 `check` 的模块,因此插件以进程的权限执行。只配置你信任的插件 ——
  把 `.octoinspect.json` 里的 `plugins` 当作任何其他代码依赖来对待。
- **发现项可能回显敏感内容。** 一个发现项可能引用一小段**脱敏后 (redacted)** 的
  违规行摘录(密钥在进入报告前已被脱敏),但 `json` 或 `sarif` 产物仍然包含你工作区
  的文件路径、行号与消息。请像对待任何静态分析报告一样对待检查输出 —— 控制谁能读取
  该 CI 产物。
- **检测是启发式的,并非穷尽。** 内置规则标记治理漏洞的常见形态;一次干净的运行并
  非安全的证明,一个发现项也并非可被利用的证明。Inspect 是其中一道闸门,而非整道
  围栏。

本项目处于 1.0 之前;仅最新版本获得修复。
